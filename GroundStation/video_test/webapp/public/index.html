WildBridge Video and Connection Testing Tool

Discovery, telemetry health, WHIP/WHEP video, and connection diagnostics.

How To Use
  1. Connect the host and phones to the same WildBridge Wi-Fi network.
  2. On each phone, enable wireless ADB if needed, then run adb connect PHONE_IP:5555 from the host.
  3. Launch the WildBridge app and set a unique drone name such as mini1, mini2, or the aircraft label you want to test.
  4. Start this stack with docker compose -f compose.video-test.yaml up -d --build and open this page.
  5. Use Discover to re-scan. A tile appears for each detected phone. Telemetry should turn green before the video path becomes ready.
  6. For drone-free video testing, enable mock video in the WildBridge Default Layout on each phone. The phone will publish the bundled clip through the normal WHIP path.
  7. Logs are written on the host under GroundStation/video_test/logs/; compare those with Android logcat and MediaMTX logs when investigating FPS drops or decode errors.
